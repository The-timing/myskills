# 移动端车牌专用键盘

## 基本规则

- 车牌号输入必须使用页面内专用键盘；不要使用 `<input>`、`textarea` 或调用系统默认键盘。
- 点击车牌槽位时打开键盘，点击遮罩或“完成”关闭键盘。
- 使用 8 个车牌槽位，支持普通车牌和新能源车牌；最终长度和格式按当前业务规则校验。
- 第 1 位只能从省份、特殊地区或特种车标识中选择；后续位使用字母数字键盘。
- 键盘必须有“省份/中文”“ABC”“删除”“完成”操作，并禁止在首位选择字母数字。
- 输入、替换和删除后同步更新当前槽位与键盘模式；车牌统一转大写。
- 不把车牌输入逻辑拆成无必要的通用层；优先在页面或独立车牌组件内保持完整状态。

## 状态模型

保持以下最小状态：

```js
licenseNumber: '',
plateKeyboardVisible: false,
plateKeyboardMode: 'province', // province | letter
activePlateIndex: 0
```

计算属性：

- `plateSlots`：将车牌拆成 8 个展示槽位。
- `currentPlateIndex`：当前输入位置，最大为 7。
- `currentKeyboardRows`：根据模式返回省份键盘或字母数字键盘。

## 交互流程

1. 打开键盘：默认定位到当前车牌末尾；用户点击已有槽位时可定位到该槽位。
2. 首位：显示省份键盘；选择省份后自动切换字母数字键盘。
3. 后续位：选择字符后写入当前槽位，最多 8 位并自动移动到下一位。
4. 删除：删除当前槽位字符，光标回退；删除为空时切回省份键盘。
5. 完成：仅关闭专用键盘，不修改已经输入的车牌。
6. 提交：统一 `trim().toUpperCase()`，再按业务要求验证 7 位或 8 位车牌。

## 实现边界

- 页面使用 `view`、槽位和自定义底部键盘渲染，避免原生输入框获得焦点。
- 键盘遮罩层级高于页面内容；键盘内部点击必须阻止冒泡，防止被遮罩关闭。
- 同一页面不得混用默认输入法和专用车牌键盘。
- 后端只接收归一化后的车牌字符串；键盘展示状态不传给后端。
