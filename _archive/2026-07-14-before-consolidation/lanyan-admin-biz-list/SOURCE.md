# LanYan 后台业务列表页技能

本技能用于在 LanYan 项目中快速生成或重构“业务型后台列表页”。

适用页面通常包括：
- 订单
- 售后
- 提现
- 佣金
- 洗衣卡订单
- 用户资产记录
- 其他以“查询 + 状态筛选 + 详情查看”为主的后台页

## 何时使用
当用户提出以下需求时优先使用：
- “整理下这个页面”
- “状态改成 tabs”
- “追加用户筛选”
- “详情改成弹窗/抽屉”
- “列表字段太多，聚合下”
- “去掉多余筛选，只保留状态和用户”

## 不适用场景
- 纯主数据维护页，重点是新增/编辑表单
- 主从编辑页，重点是 tabs 子表维护
- 可视化大屏、首页仪表盘

## 标准处理方式

### 1. 先判断页面类型
- 如果页面是浏览/审核/记录查看页，优先收敛成只读页
- 如果页面是主数据维护页，再保留新增、修改、删除

### 2. 状态筛选优先改为顶部 tabs
- `全部` 必须使用显式值 `all`
- tabs 选中值与 `queryParams.status` 分离维护
- 切换 tabs 时做 `all -> null` 映射

### 3. 用户筛选优先远程搜索
- 默认补一批用户选项
- 页面创建时主动加载默认数据
- 下拉展开无数据时自动加载
- 搜索词清空后恢复默认选项
- 搜索字段至少支持：
  - 用户ID
  - 昵称
  - 姓名
  - 手机号

### 4. 列表字段优先做聚合列
- 不要平铺大量裸字段
- 常见聚合块：
  - 基础信息
  - 用户与关联信息
  - 金额与状态
  - 时间信息
- 单元格内使用多行 `div`
- 表头保持居中，内部内容左对齐

### 5. 详情优先用 dialog 或 drawer
- 优先使用 `el-dialog` 或 `el-drawer`
- 主信息用 `el-descriptions`
- 关联列表用内嵌 `el-table`
- 空值统一显示 `--`

### 6. 用户与外键优先后端回显
- 列表里不要默认展示裸 `userId`、`schoolId`、`productId`
- 优先在 `Vo` 中补扩展字段
- 由后端 service 聚合注入昵称、姓名、手机号、学校名、商品摘要等

### 7. 只读页要真实收敛 CRUD
- 前端删除无关按钮和无关提交逻辑
- API 文件清理不用的新增/修改/删除方法
- 后端控制器同步收敛不需要的写接口

## 推荐输出结构

### 前端页面
- 搜索区：少而关键
- 状态 tabs：顶部主筛选
- 表格：聚合列
- 详情弹窗：分块展示

### 前端方法
- `getList`
- `handleQuery`
- `resetQuery`
- `handleStatusTabChange`
- `getDefaultUserOptions`
- `remoteUserMethod`
- `handleView`

## 最小实现清单
- 一个 `activeStatus`
- 一个 `statusOptions`
- 一个远程用户筛选
- 一个聚合列模板
- 一个详情弹窗
- 一个重置逻辑

## 验证清单
- tabs 是否只有一个高亮
- 重置后 tabs 与 queryParams 是否一致
- 用户筛选是否有默认数据
- 清空搜索词后是否恢复默认用户
- 列表是否仍有过多裸字段
- 详情是否按业务块分段展示
- 页面是否仍残留无用 CRUD 代码
