# LanYan Frontend View Generator

本技能用于生成或重构符合 LanYan 项目前端 Vue 组件规范的代码。

## 8. 前端开发模式

### 8.1 一对多关系 (主从) 模式
- **场景**: 编辑包含子实体列表的实体时（例如 订单 -> 订单项, VIP -> VIP项）。
- **UI 模式**: 在 `el-dialog` 中使用 `el-tabs` 将基本信息与子列表分开。
- **结构**:
  - **Tab 1 (基本设置)**: 包含主实体属性的 `el-form`。
  - **Tab 2+ (子列表)**: 包含管理子列表的独立组件。
- **实现**:
  - 通过 `props` 将子列表数组从父表单传递给子组件。
  - 监听来自子组件的更新事件（例如 `@backList` 或 `@update:list`）以更新父表单数据。
  - 如果需要，使用 `:key` 强制重新渲染（例如 `:key="Math.random()*10000"`）。

## 10. 二值字段 UI 规范 (el-switch)
- **适用范围**: 取值仅有两种状态（如 0/1、true/false、启用/停用、展示/不展示）的字段。
- **列表展示与操作**:
  - 在 `el-table` 中使用 `el-switch` 直接进行切换操作。
  - 使用 `:active-value="1"`、`:inactive-value="0"` 映射后端枚举值。
  - 在 `@change` 事件中调用更新接口，仅提交变更字段与主键。
- **表单编辑**:
  - 在 `el-form-item` 中使用 `el-switch`，并设置 `active-text` / `inactive-text` 以增强可读性。

## 11. 外键展示渲染规范

- 当页面需求是 `名称展示`、`tag 展示`、`详情回显`、`摘要展示` 时，前端不要直接渲染裸 `xxxId`、`xxxIds`。
- 如果接口只返回外键 ID，而页面又需要名称、标题、手机号、商品信息等业务可读内容，优先推动后端补齐聚合字段，不在前端临时拼装主展示信息。
- 前端推荐用法：
  - 裸 ID 用于提交、跳转、回填选择值
  - 聚合字段用于列表列、`el-tag`、详情面板、摘要区渲染
- 如果是多选外键场景，例如 `recommendProductIds`，优先消费后端回传的对象列表，再统一渲染为标签组，不自己拆字符串做主要展示。

## 12. Vue2 编辑表单回填规范

- 在 Vue2 页面中，编辑弹窗优先先用 `reset()` 声明完整表单结构，再做详情回填。
- 对包含 `多选数组`、`已选对象列表`、`级联路径`、`标签区` 的表单，不要直接 `this.form = response.data` 整对象替换。
- 推荐写法：
  - 保留原有 `this.form` 响应式对象
  - 使用 `Object.assign(this.form, detail, extraFields)` 合并详情数据
  - 把 `recommendProductList`、`relatedProductList`、`categoryPath` 这类模板依赖字段预先声明在默认表单中
- 如果页面存在“数据已经变化但标签区不刷新”的现象，优先检查是否出现了“整对象替换后再动态补字段”的写法。
