# 知识条目目录说明

本目录只存放标准化知识条目。

## 1. 存放规则

- 一条知识对应一个独立问题场景
- 文件必须使用 Markdown
- 文件必须从 `TEMPLATE.md` 生成
- 文件名建议使用：
  - `KB-[分类编码]-[8位递增序号]-slug.md`

## 2. 内容约束

- 不允许存放项目专属业务代码
- 不允许省略必填字段
- 不允许自定义非标准标签
- 不允许把多个无关问题混在一条知识中

## 3. 入库前检查

- ID 是否符合规则
- 标签总数是否不超过 8 个
- 技术栈标签、问题类型标签、场景标签是否齐全
- 是否写明触发条件、影响范围、复现步骤、根因、方案、验证、更新记录、关联条目

## 4. 生成方式

新增知识条目时，优先使用 `../scripts/new-entry.ps1` 基于 `TEMPLATE.md` 生成，再按真实问题补全内容。
