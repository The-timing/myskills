# 通用知识库桥接规则

## 触发条件
全局生效。默认启用 `.trae/kb` 作为共享知识库入口。

当处理本项目中的需求、Bug、重构、页面治理、接口治理、状态治理、链路一致性问题时，必须先检查 `.trae/kb` 是否已有可复用知识；只有在明确判断为“纯当前项目业务口径问题”且不具备通用复用意义时，才允许跳过知识库检索。

## 详细要求

### 1. `.trae/kb` 不是普通文档目录，而是优先检索源

**内容**:
当问题属于“通用开发场景”或“历史上可能重复出现的处置模式”时，不能只看当前项目 `.trae/rules`，还必须先检查 `.trae/kb` 是否已有可复用知识。

**默认规则**:
- 默认先查 `.trae/kb`
- 默认优先复用已有条目
- 默认在“项目规则 + 通用知识”双重约束下生成方案
- 只有在问题明显属于当前项目专属业务解释时，才可不命中 `.trae/kb`

**适用问题**:
- 架构与抽象问题
- 状态流转问题
- 接口边界问题
- 预览与执行一致性问题
- 后端兜底问题
- 返回结构设计问题
- 后台列表治理问题
- 远程筛选与聚合回显问题
- 测试流程与正式流程一致性问题
- 资源占用与释放问题

### 2. `.trae` 与 `.trae/kb` 的职责必须区分

**内容**:
- `.trae/rules`：当前项目的业务规则、状态口径、接口边界、页面治理规范
- `.trae/kb`：跨项目可复用的通用处置经验与解决模式

**规范**:
- 不允许把项目专属业务规则误当成 `.trae/kb` 通用知识
- 不允许只用 `.trae/kb` 替代当前项目 README 和业务规则
- 必须先判定“这是项目规则问题”还是“这是通用处置问题”

### 3. 处理问题时的标准顺序

**内容**:
处理大多数开发问题时，按以下顺序执行：

1. 读取项目根目录 `README.md`
2. 读取 `.trae/README.md`
3. 根据主线进入 `.trae/rules/scene-index.md`
4. 如果问题跨多个对象、状态或处理环节，读取 `.trae/rules/scene-handling-framework.md`
5. 默认检索 `.trae/kb`
6. 如果命中通用知识，则优先复用
7. 如果未命中，再基于当前项目规则补充方案
8. 最后输出“项目规则 + 通用知识”联合收敛后的实现方案

### 4. `.trae/kb` 的检索入口

**内容**:
检索 `.trae/kb` 时，优先使用以下文件：

- `.trae/kb/maintenance/generated/knowledge-index.json`
- `.trae/kb/retrieval/query-template.md`
- `.trae/kb/retrieval/match-rules.md`
- `.trae/kb/retrieval/tag-dictionary.yaml`

**规范**:
- 先根据问题摘要、错误关键词、场景标签整理检索输入
- 再从 `knowledge-index.json` 中定位最相似条目
- 命中后优先复用已有条目，不要重复发明同类方案

### 5. 对话中的默认动作

**内容**:
当用户没有特别说明时，也应主动执行以下动作：

1. 先说明将结合 `.trae/kb` 检索历史处置模式
2. 再结合当前项目规则收敛实现口径
3. 如果本次问题形成了新的跨项目经验，应建议沉淀到 `.trae/kb`

**默认例外**:
- 单纯业务文案修改
- 当前项目专属枚举口径解释
- 当前项目临时联调值修正
- 当前项目一次性数据修补

### 6. 何时将本次处理升级到 `.trae/kb`

**内容**:
当一次处理同时满足以下条件时，应建议入库：

- 不依赖当前项目独有业务词才能成立
- 未来其它项目大概率也会遇到
- 处置步骤稳定，可复现、可验证
- 能抽象成“问题特征 + 根因 + 处置步骤 + 验证标准”

**禁止入库**:
- 当前项目专属业务状态口径
- 当前项目专属接口路径
- 一次性脏数据修复
- 临时联调口径

### 7. 用户常用触发语义

**内容**:
以下表达默认视为要联动 `.trae/kb`：

- “先按知识库查一下”
- “先看看以前有没有类似处理”
- “这个问题值得沉淀”
- “处理完升级进知识库”
- “不要从零分析，先复用历史方案”

### 8. 验证清单

每次处理复杂问题前至少检查：
- 是否已经完成 README 和项目规则定位
- 是否已经判断该问题是否属于可复用通用场景
- 是否已经检查 `.trae/kb` 是否存在相似知识条目
- 最终方案是否同时满足“项目正确性”和“通用复用性”
- 本次结论是否值得沉淀到 `.trae/kb`
