# 蓝燕管理后台

## 设计操作流程

1. 从操作员的高频任务出发：筛选、浏览、查看、编辑、处理或删除。
2. 列表列展示可读的业务信息；ID 只作为辅助追踪信息。
3. 将密集或多字段信息放入字段分组清晰的详情弹窗/抽屉。
4. 只保留当前业务状态和权限允许的操作；移除过期的编辑/删除控件，不保留无效按钮。
5. 从真实数据源加载筛选候选项，并在清空关键词后恢复合理默认值。

## 通用布局规则

- 状态是主要工作划分时使用状态标签页。
- 两级分类以一级作为选择面板、二级作为工作面板；仅在拥有图片的层级显示图片控件。
- 成员/团队页必须使用选中成员 ID 打开详情；不得复用当前管理员身份或海报。
- 菜单、路由、按钮权限和后端接口变更必须同步。

## 数据契约

- 在后端 VO 中补展示字段，不在浏览器端解析外键。
- 列表和详情查询返回页面实际需要的字段。
- 保持 Vue2 表单响应性：涉及选中状态时填充既有响应式字段，不替换整个表单对象。

## 验证

检查首次加载、筛选、空状态、清空/重置、详情打开、允许操作、权限隐藏操作、新建/编辑持久化和刷新。
